# Інструкції для Claude - Frontend (Vue)

## Мова спілкування
- Завжди відповідай українською мовою
- Всі пояснення, описи та текстові відповіді мають бути українською

## Правила роботи з кодом
- НІКОЛИ не додавай коментарі в код
- Не залишай жодних пояснювальних коментарів при написанні нового коду
- Не додавай коментарі при редагуванні існуючого коду
- Код має бути чистим, без коментарів
- Всі пояснення про код надавай у відповіді, а не в коментарях коду

## Кодування файлів
- Завжди використовуй UTF-8 кодування для всіх файлів
- Переконайся, що українські символи відображаються коректно
- При створенні або редагуванні файлів зберігай їх у UTF-8

## Робота з Tailwind CSS
- Завжди використовуй Tailwind CSS для стилізації компонентів
- Не створюй окремі CSS файли або `<style>` блоки
- Використовуй utility-класи Tailwind безпосередньо в шаблонах
- Для складних стилів використовуй композицію Tailwind класів
- Дотримуйся consistency у використанні spacing, colors та typography з Tailwind

## Архітектура компонентів Vue
- Максимально розбивай UI на окремі компоненти
- Створюй невеликі, переглядні компоненти з однією відповідальністю
- Витягуй повторювану логіку та розмітку в окремі компоненти
- Дотримуйся принципу DRY (Don't Repeat Yourself)
- Використовуй Composition API для нових компонентів
- Структуруй компоненти логічно: pages/, components/, composables/
- Кожен компонент має бути переглядним (до 200 рядків коду)
